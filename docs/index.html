autocomplete v3.0.3

    Getting started

    Install via npm

    If you use npm or yarn, you can install the library with the following command:

                  npm install @tomickigrzegorz/autocomplete

    Or using Yarn: 

                  yarn add @tomickigrzegorz/autocomplete

    Include JS and CSS files

    The first thing you should do is download the autocomplete library files. All the necessary files are located in the "dist/js" and "dist/css" folders. If you want to make changes, the uncompiled javascript and sass files are in the sources folder. 

    <!-- css -->
<link rel="stylesheet" href="path/to/autocomplete.min.css">

<!-- js -->
<script src="path/to/autocomplete.min.js"></script>

    Or CDN

    <!-- css -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/tomickigrzegorz/autocomplete@3.0.3/dist/css/autocomplete.min.css"/>

<!-- js -->
<script src="https://cdn.jsdelivr.net/gh/tomickigrzegorz/autocomplete@3.0.3/dist/js/autocomplete.min.js"></script>

    Cache data

    source: cache
    The fetchDataWithSessionCache function implements caching of search results in sessionStorage, which speeds up data loading and reduces the number of API requests.

    1. CACHE_TTL: Defines the cache's time-to-live (TTL) as 60 seconds.
    2. Cache Check:
      1. The function checks if the query (query) exists in sessionStorage and if it is not outdated (i.e., not older than 60 seconds).
      2. If the data is still valid, it is returned directly from the cache.
    3. Fetching Data from the API:
      1. If the data is not in the cache or is outdated, the function makes a request to the API (in this case, the "Rick and Morty" API).
      2. The retrieved results are saved to sessionStorage with the current timestamp.
      Open devTools in chrome. Find Application -> Session storage and see how the data is saved. Of course, the data is only in the session, so if I close the tab, the data is deleted. You can change it to Local storage or even save it in memory, but it's up to you how you save the data.

    Disable Autocomplete

    source: disable

    The disable method can be called to disable the autocomplete functionality. When disabled, the autocomplete will not respond to user input or show suggestions. This can be useful in scenarios where you want to temporarily prevent user interaction with the autocomplete feature without destroying the instance entirely.

    Enable/Disable Toggle

    source: enable

    The enable() method restores autocomplete functionality after it has been disabled using disable(). This allows you to toggle autocomplete on/off while preserving the same instance and configuration.

    Enable/Disable with Empty Input

    source: enable-disable

    This example shows how autocomplete transforms a regular input field. Initially the input is completely normal - no ARIA attributes, no autocomplete functionality.

    • Enable: Creates autocomplete instance and adds ARIA attributes
    • Disable: Removes functionality but keeps the instance
    • Fill Input: Adds random data (triggers search only if enabled)

    Watch the console and inspect the input element to see the transformation!

    Show inline

    source: update-input

    Adding this parameter adds the selected data to the input field while navigating through the records with the up/down arrows or hovering with the mouse. insertToInput: true

    From now when insertToInput: true the input value will be updated automatically after hovering on an item from the list. But if you want to modify the input value, you can use the 'onSelectedItem' event. onSelectedItem: ({ element, object }) => {element.value = object.name}
